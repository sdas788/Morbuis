## Steps
Send/Receive Alert – Happy Path (Trigger → Receive → Open)

## Expected Result
Precondition: user is logged in, OS notifications allowed, in-app alerts enabled, and hub/sensor is configured. Trigger a test alert event (e.g., force a sensor value to breach a critical threshold using test tooling/simulator). User receives the alert (push notification and/or in-app alert banner depending on app behavior). The alert appears in the Alerts/Notifications list with correct hub/sensor name, severity/type, and timestamp. Tapping the notification (or the alert in the list) opens the app to the correct hub/sensor detail context (or alert detail screen), without errors.
