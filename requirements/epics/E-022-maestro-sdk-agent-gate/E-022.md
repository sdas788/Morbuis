## ⚠ This epic is a Decision Gate — no build

The only deliverable is a written decision in `requirements/decisions/` explaining whether to build a standalone Claude Agent SDK (or OpenAI SDK) agent that wraps the Maestro skills we've shipped inside Claude Code.

**Recommendation: DO NOT BUILD** unless one of these gate criteria is met:

1. A concrete capability the in-Claude-Code skills cannot reach (e.g., unattended scheduled runs outside an IDE session).
2. A customer or partner requirement for a standalone agent binary.
3. Maestro partnership formalizes and requires a shipped integration artifact.

Until a gate fires, close this epic with a decision doc and move on.

## Why this is a gate, not a build

The current Maestro MCP skills already work inside Claude Code. Rebuilding them as a standalone SDK agent without a concrete user problem is a rebuild-for-rebuild risk — classic architecture astronautism that burns weeks and delivers no user value.
