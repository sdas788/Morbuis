## Outcome

Morbius can execute test cases against web applications, not just mobile. Each project declares its `projectType` (`mobile` / `web` / `api`); web projects gain a `webUrl` and run their tests via an agent-orchestrated browser-MCP runner. Two backends ship in v1: **headless** (Claude CLI + Playwright MCP) for automation, **visual** (Claude in Chrome) for human-watching debug runs. Test cases stay declarative — natural-language Given/When/Then transferred from PMAgent — so the same runner contract works for any future agent (Codex, etc.) with a browser MCP.

The validation walkthrough is the dogfood scenario: open the `morbius` project (200 transferred test cases), set `projectType: 'web'`, run a Jira-sync test against `http://localhost:9000`, watch Claude drive headless Chromium, get pass/fail with screenshots back. **Morbius testing Morbius** is the proof.

A production-deployment trajectory is captured in arch.md so the eventual swap from `claude --print` subprocess (today, dev-laptop) to **Claude Agent SDK** (containerized server) and later **Anthropic Managed Agents** (multi-tenant scale) is a one-file change, not a rewrite. Direction Guardrail #5 still binds in v1: no Agent SDK adoption until E-022 gate criteria are met.
