## Outcome

A user opens Morbius's AppMap tab and immediately understands: *why these specific N flows are automated out of the full test catalog, what the agent learned along the way, and what's happening in runs.* The Mermaid chart stays — but it's now visually first-class (designer-grade theme, status-coded nodes, hover affordances) and is paired with a Claude-generated narrative panel and a per-flow accordion with run insights.

E-027 sits **next to E-018** (the recommender). E-018 answers "what should we automate next?" — E-027 answers "why are we automating what we're automating?"

## Success Metric

| Metric | Baseline | Target |
|--------|----------|--------|
| User opens AppMap tab and can answer "why these flows" without asking | 0 / 5 RF QA | 5 / 5 within 1 demo session |
| Per-flow rationale references specific flow files (not boilerplate) | n/a | ≥90% of `whyPicked` entries pass lint |
| AppMap chart visual upgrade lands at "designer-grade" bar | functional default Mermaid | matches Linear/Vercel/Notion polish (subjective; verified by side-by-side review) |

## Constraints

- **C1. Retrospective only.** No live exploration agent in this epic. Claude reads what we already have (flows, test cases, AppMap, run history) and writes the narrative. A live-crawl variant is a future epic if retrospective proves insufficient.
- **C2. Reuse the E-016 pattern.** Frontmatter markdown file under `data/{projectId}/`, typed read/write in `parsers/markdown.ts`, generated via `askClaude()`. No new infrastructure.
- **C3. Stay on Mermaid.** Visual upgrade lives inside Mermaid theming + classDef + post-render SVG decoration. ReactFlow / D3 swap is a separate decision deferred unless theming hits a hard wall.
- **C4. Cloud-deploy clean.** All artifacts under `MORBIUS_DATA_DIR` (E-025). Nothing baked into the image; nothing Fly-specific in this epic.
