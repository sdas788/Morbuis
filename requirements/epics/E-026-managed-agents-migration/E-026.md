## Outcome

The v2.0 cloud Morbius runs Claude CLI inside its container — Morbius operates a subprocess, manages its lifecycle, retries, observability. v2.1 flips that: the agent becomes a **task definition Morbius posts to Anthropic's hosted runtime** (Managed Agents). Morbius stops being a process supervisor and starts being a job submitter + result reader.

**Why:** zero CLI/SDK process management in our Fly container, Anthropic handles scale + retries + multi-region, agent reliability decouples from our infra. **Risk:** the browser-MCP-locality problem (Constraint C6 from E-025) — the managed agent runs in Anthropic's infra and can't reach a Playwright MCP server in our Fly container. The first story is a 1-week derisking spike to pick the architecture before any commitment.

## Success Metric

| Metric | v2.0 Baseline | v2.1 Target |
|--------|---------------|-------------|
| Container memory footprint (steady state) | ~1.5 GB (Chromium + CLI) | <800 MB (no CLI, no embedded browser if Computer Use API picked) |
| Agent failure recovery time | manual / restart container | Anthropic-managed retries, no human |
| Concurrent test runs supported | 1 (single VM) | 5+ (Anthropic handles concurrency) |
| Cost per agent-minute | Anthropic API only | Anthropic API + (small) Managed Agents premium — measure during spike |

## Constraints (must read before implementation)

- **C1. Browser-locality is non-negotiable.** Every web test in Morbius drives a real browser via Playwright MCP. If the v2.1 architecture cannot reach a browser, this migration fails and we stay on v2.0 indefinitely. S-026-002 picks the answer; S-026-003 doesn't start until S-026-002 lands.
- **C2. CLI mode stays in production for one full release cycle after cutover.** Defense in depth — if Managed Agents misbehaves, flip the env var back, no redeploy. The chokepoint at `src/runners/web-agent.ts` already supports both modes side-by-side.
- **C3. No regression on user-visible behavior.** Test results, screenshots, run logs, SSE chat — all behave identically before and after cutover. The migration is a runtime substitution, not a UX change.
- **C4. Cost ceiling.** Managed Agents pricing TBD as of 2026-04-30. The spike measures actual per-run cost. If it's >2× our v2.0 Anthropic API spend, we either negotiate or stay on v2.0 — we do not silently double infra cost.
- **C5. Direction Guardrail #5 stays current.** v1.1 addendum (S-025-006) names Managed Agents as the v2.1 target with Agent SDK as fallback. If S-026-002 picks the hybrid worker option, the addendum must be revised again so future-you sees the actual architecture.
