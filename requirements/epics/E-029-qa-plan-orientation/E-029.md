## Backfill Note

This epic documents the largest block of untracked work found in the 2026-06-10 drift audit: the QA-team orientation layer built on top of the test cases. The code referenced "E-029" with no epic behind it. Created retrospectively; all stories are already Done and ACs describe shipped behavior.

## Outcome

PMAgent is the **product** board; Morbius is the **QA** board. Before this epic, a QA tester opening a freshly-synced project saw a grey wall of raw test cases — no roadmap, no honesty about coverage. This epic brings the product team's roadmap *above* the cases (the QA Plan, Flow Plans, Release Plans, and User Flows — copied verbatim on a one-click push and rendered Morbius-native), then makes the board **legible and honest**: it quarantines meta/raw epics, surfaces source drift, shows true coverage, and adds the truth PMAgent's QA tab can't — *does the plan actually pass?* PMAgent owns "planned"; Morbius owns "passing."
