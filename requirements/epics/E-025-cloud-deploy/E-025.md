## Outcome

Any RF team member opens `https://morbius.redfoundry.dev`, signs in with their `@redfoundry.com` Google Workspace account via Cloudflare Access, and lands on a shared QA dashboard with all the team's projects, test cases, and run history. They can run web test cases against any deployed RF client web app from there. **Goal: get Morbius off "Saurabh's laptop" and onto a URL the team can use.**

v2.0 ships **web-only** in cloud (mobile testing stays a local-laptop tool). Single shared workspace for the RF team — not multi-tenant. Container has Claude CLI baked in for the agent runtime; the migration to **Anthropic Managed Agents** is the v2.1 north star (E-026), gated by a 1-week derisking spike.

## Constraints (must read before implementation)

These are real architectural constraints baked into the v2.0 plan. Each one informs at least one story's scope:

- **C1. Cost.** Fly.io ~$5–10/mo for a 1 GB VM + 10 GB volume; Chromium under load could push to 2 GB. Total infra <$30/mo. Cloudflare Access free tier covers 50 users. Anthropic API costs are the variable — measure during the first week.
- **C2. Single shared workspace.** Everyone on the RF team sees and edits the same projects. Last-write-wins on the markdown writers. Race conditions possible but acceptable for v2.0; revisit if it bites.
- **C3. PMAgent stays separate.** Cloud Morbius reads from a server-side git checkout (S-025-005). Authoring still happens locally. The Transfer / Publish round-trip works in cloud as long as the bootstrap script keeps the checkout fresh.
- **C4. Mobile cloud parity is deferred.** Hosted Morbius's mobile projects will be partially functional (visible, with run history) but Run buttons error. UX should communicate this with a banner. v2.2 fixes it via a `morbius-runner` local agent companion.
- **C5. Anthropic CLI is moving.** Pin `@anthropic-ai/claude-code@1.x.x` in the Dockerfile. Bumps are explicit, reviewed, deployed.
- **C6. Managed Agents browser-locality problem (the central v2.1 risk).** When v2.1 moves to Anthropic Managed Agents, the agent runs in Anthropic's infra — it can't reach a Playwright MCP server hosted in our Fly container. Three options: (a) **Anthropic Computer Use API** (virtual desktop the managed agent drives), (b) **expose Playwright MCP publicly** with auth (DDoS exposure), (c) **hybrid** (managed orchestration + local worker for browser actions). E-026's first story is a 1-week spike to pick.
